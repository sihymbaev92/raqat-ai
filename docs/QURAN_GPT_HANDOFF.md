# RAQAT жобасы: Құран мәтіні / транскрипция — GPT үшін қысқа бриф

**Толық платформа (бот, API, веб, мобильді):** `docs/PLATFORM_GPT_HANDOFF.md`

**Ресми дерек көздері (жобада осыны басшылыққа аламыз):**

1. **Қазақша мағына** — Ерлан Алимулы тексерілген баспа: `data/quran_kk_verified.json` (6224 аят, репоға кірмейді) → `bash scripts/run_erlan_quran_import.sh` немесе `scripts/import_quran_kk_verified.py` (`--attribution-kk` / `--source-detail` опциялары бар).
2. **Транскрипция** — дерекқорда **бір бағана** (`translit`); толтыру **бір реттік ретпен**: алдымен [koran.kz/trnc/](https://koran.kz/trnc/) (`import_quran_translit_koran_kz.py`), **бос қалған** аяттарға ішкі алгоритм (`backfill_quran_translit.py` → `transliterate_arabic_to_kazakh`). Бір реттік синх: `bash scripts/sync_quran_official_sources.sh` (коран импорты осының 1/2 бөлігі).

Бір ретте екеуін орындау: `bash scripts/sync_quran_official_sources.sh` (алдымен `--dry-run` тексеріңіз). `DB_PATH` орта айнымалысымен басқа sqlite көрсетуге болады.

Бұл құжатты басқа модельге жіберуге болады: репозиторийдегі **нақты не бар**, **қалай жұмыс істейді**, **қай көздер аталады**, **қазіргі күй** қысқаша сипатталады. `.env` құпиялары осы файлға енгізілмейді.

| Бағана | Мазмұны |

## 2. Мағына (`text_kk`) — қайдан және қалай импорт

- **Тексерілген баспа** ретінде жобада мысал ретінде **Ерлан Алимулы** аудармасы аталады. Файл: `scripts/import_quran_kk_verified.py` — JSON пішімі докстрингте; `quran.text_kk` жаңартады, `quran_kk_provenance` кестесіне `attribution_kk`, `source_detail` жазады.
- **qurankarim.kz API** арқылы импорт: `scripts/import_quran_kk_qurankarim.py` — сайт meta бойынша **Халифа Алтай** аудармасы екені ескертіледі (Ерлан Алимулы осы API-да көрсетілмейді).
- **Gemini** арқылы толтыру (діни мәтін үшін сақтандырумен): `translate_quran_kk_batch.py` — тек техникалық тест / бос орын; промптта 1-сүре стиль үлгісі бар. `GEMINI_API_KEY` қажет.

## 3. Транскрипция (`translit`)

**Дерекқорда көздер «үш бөлек API» емес:** `translit` — **бір бағана**, ал мән **ресми толтыру реті** бойынша қалыптасады (көрінбейтін «бір ақ дерек» — осы схема).

### 3.0 Канондық толтыру реті (негізгі)

1. **`scripts/import_quran_translit_koran_kz.py`** — [koran.kz/trnc/](https://koran.kz/trnc/) (Falaq жобасына сілтеме бар). Сайтта қазақ кирилл мәтіні бар аяттарға жазады; **ұзын сүрелерде** сайтта кирилл толық болмауы мүмкін — сол аяттарда бағана бос қалады немесе бұрынғы мән сақталады (скрипт докстрингіне қараңыз).
2. **`backfill_quran_translit.py`** — әлі де **бос** `translit` жолдарын `services/quran_translit.py` → `transliterate_arabic_to_kazakh()` арқылы толтырады (`--force` — бар мәнді қайта есептеу). Осылайша **6224 аяттың барлығы** мәтінмен қамтамасыз етіледі.
3. **`scripts/import_quran_translit_json.py`** — **негізгі жол емес**; тек расталған JSON-мен **түзету**, баспа нұсқасынан көшіру немесе миграция үшін.

Бір командада коран импорты + (қолда бар болса) мағына: `bash scripts/sync_quran_official_sources.sh`.

### 3.1 Ішкі алгоритм (`backfill` қолданады)

- Файл: `services/quran_translit.py` — функция `transliterate_arabic_to_kazakh(...)`.
- Бұрынғы сигнатурадан кейін: `surah`, `ayah`, `style` опционалды.
- **`style="pedagogical"`**: 1-сүре 1–7 аят үшін кодта тұрақты **пайдаланушы үлгісі** (`FATIHA_PEDAGOGICAL_PRESET`); қалған аяттар үшін әдепкі нәтиже + тиреді бос орынға жақындату (`_pedagogical_spacing`).
- Баптау: `config/settings.py` → `QURAN_TRANSLIT_STYLE` = `default` \| `pedagogical` (`.env`).

### 3.2 Дерекқорды жаңартатын скрипттер (рөлдері)

| Скрипт | Рөлі |
|--------|------|
| `scripts/import_quran_translit_koran_kz.py` | **1-кезең:** koran.kz кирилл (жобадағы негізгі сыртқы көз). |
| `backfill_quran_translit.py` | **2-кезең:** бос жолдарды алгоритммен толтыру; `--force` — қайта есептеу. |
| `scripts/import_quran_translit_json.py` | **Қосымша:** түзету JSON; күнделікті толтыру үшін міндетті емес. |

### 3.3 Тексеру / аудит

| Скрипт | Не істейді |
|--------|------------|
| `scripts/validate_quran_translit.py` | `translit` ішінде араб Unicode / латын a–z қалдықтары |
| `scripts/quran_translit_status.py` | Бос жол саны, алгоритммен сәйкес келмеу саны |
| `scripts/audit_quran_translit.py` | Сол метрикалар + опция `--list-mismatch N` |
| `tests/test_quran_translit.py` | Алгоритм юнит-тесттері |

## 4. Ботта көрсету

- Файл: `handlers/quran.py` — `_get_surah_chunks` сұранысына `surah` бағанасы қосылған; `QURAN_TRANSLIT_STYLE=pedagogical` болса, транскрипция бөлімі **дерекқорды елемей** сол режим бойынша есептеледі (1-сүре эталон + қалғаны spacing); әйтпесе `translit` бос болса алгоритм шақырылады.

## 5. Қазіргі типтік күй (сонғы тексеруден кейінгі үлгі)

- **6224** жол; **бос `translit` жоқ** (канондық рет: koran.kz → содан кейін `backfill`).
- **`validate_quran_translit.py`**: араб/латын қалдық — **0**.
- **`quran_translit_status.py`**: кей аяттар алгоритм «күткен» нәтиге мен сәйкес келмеуі мүмкін — себебі **koran.kz** осы аяттарға бөлек кирилл берген; бұл қате емес, **көздер біріктірілген** схеманың күтілетін салдары. Ұзын сүрелерде koran.kz кирилл аз болса, сол сүрелерде `translit` негізінен алгоритмнен.

## 6. Салыстыруға арналған «баспа» репода бар ма?

**Жоқ** — дайын PDF/кітап файл жоқ. Тек **сыртқы көздер** көрсетілген:

- Мағына: **Ерлан Алимулы (тексерілген баспа)** — лицензиямен JSON импорт;
- Сайт: **qurankarim** / **Халифа Алтай** (скрипт ескертпесі);
- Транскрипция веб: **koran.kz/trnc**.

## 8. GPT-ке тапсырма мысалы

> SQLite `quran` кестесінде 6224 аят бар. `translit` — **бір бағана**, толтыру **бір реттік**: koran.kz (болғанда), содан кейін бос қалғанға алгоритм; JSON/қолмен түзету — ерекше. Кей аяттар koran.kz кирилліне сәйкес; кейбіреулері толығымен алгоритмдік. Мен сенен: (а) сыртқы тексерілген баспа/аудио стандартымен салыстырғанда қай аяттарды алдымен адам қарауы керек екенін жоспарлау; (б) JSON импорт пішімін ұсыну; (в) стильді біріктіру стратегиясын ұсыну — деп сұрауға болады.
