# Сахих хадис `text_kk` — редакциялық батчтар (db `hadith.id`)

SQLite кестесінде **кітап нөмірі жеке баған емес**; жұмысты **дерекқор id ауқымы** бойынша бөлу ыңғайлы. Төмендегі ауқымдар `global_clean.db` үлгісіне сәйкес (өзгерсе `MIN(id)` / `MAX(id)` қайта алынсын).

## Дерекқор ауқымы (мысал)

| Жинақ | `source` | min id | max id | шамамен саны |
|-------|----------|--------|--------|--------------|
| Сахих әл-Бұхари | `Sahih al-Bukhari` | 101223 | 108499 | 7277 |
| Сахих Муслим | `Sahih Muslim` | 108500 | 115867 | 7368 |

## Бұхари — батчтар (≈500 id / топ)

Редакция кезегін белгілеу үшін (id қоса алғанда):

| Батч | id ауқымы | Ескерту |
|------|-----------|---------|
| B-01 | 101223–101722 | алғашқы топ |
| B-02 | 101723–102222 | |
| B-03 | 102223–102722 | |
| … | … | қажет болса ұзартыңыз |
| B-15 | 107723–108499 | соңғы топ (ұзындығы <500 болуы мүмкін) |

**Классикалық «кітап 1»** (басылым бойынша) осы id ауқымына **түп-тұйық сәйкес келмеуі мүмкін** — салыстыру үшін араб мәтін мен басылымдағы нөмірді қолданыңыз; JSON экспорттағы `reference` / `dbId` — жұмыс идентификаторы.

## SQL мысалдары (SQLite)

Белгілі батш бойынша id тізімін алу:

```sql
SELECT id, LENGTH(COALESCE(text_kk,'')) AS kk_len
FROM hadith
WHERE source = 'Sahih al-Bukhari' AND id BETWEEN 101223 AND 101722
ORDER BY id;
```

Күдікті жолдарды аудит скриптімен шолу (`data/hadith_kk_repair_ids.txt`):

```bash
.venv/bin/python scripts/audit_hadith_kk_quality.py --db global_clean.db --write-ids data/hadith_kk_repair_ids.txt
```
